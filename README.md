# SPORTIK

Минималистичное приложение для домашних тренировок с гантелями и скамьёй: интерактивная техника, недельный план, прогресс, персональный AI‑тренер и синхронизация между устройствами.

## Что уже работает

- 10 упражнений с интерактивным просмотром фаз движения;
- пошаговая тренировка с фиксацией каждого подхода;
- выбор веса одной гантели и их количества (по умолчанию 2 × 5 кг);
- адаптация локального тренера по времени, усталости, боли и оборудованию;
- профиль, статистика и хранение прогресса в браузере;
- подготовленная авторизация Supabase по e-mail и Google;
- облачная синхронизация одного прогресса на компьютере и телефоне;
- защищённый серверный endpoint для OpenAI Responses API;
- PWA: приложение можно добавить на домашний экран телефона.

Без настроенных облачных сервисов SPORTIK продолжает работать автономно и хранит данные локально.

## Рекомендуемая архитектура

- **Vercel** — публичный сайт и защищённая функция `/api/coach`;
- **Supabase** — вход, база данных и Row Level Security;
- **OpenAI Responses API** — персональный тренер;
- **GitHub** — публичный исходный код и автоматическое развёртывание в Vercel.

GitHub Pages не подходит для AI‑части: на чисто статическом сайте нельзя безопасно хранить `OPENAI_API_KEY`.

## Локальный просмотр без облака

Откройте `index.html` либо запустите сервер из этой папки:

```bash
python3 -m http.server 4173
```

Затем откройте `http://127.0.0.1:4173`.

## 1. Подключение Supabase

1. Создайте бесплатный проект на [supabase.com](https://supabase.com/).
2. Откройте **SQL Editor** и выполните файл [`supabase/schema.sql`](supabase/schema.sql).
3. В **Authentication → URL Configuration** добавьте локальный и будущий Vercel URL.
4. Вход по e-mail работает после настройки Site URL. Google можно дополнительно включить в **Authentication → Providers → Google**.
5. Скопируйте Project URL и anon/publishable key. Это публичные клиентские параметры; безопасность данных обеспечивается RLS из `schema.sql`.

## 2. Подключение OpenAI

1. Создайте отдельный project API key на [странице API keys](https://platform.openai.com/api-keys).
2. Никогда не вставляйте ключ в `app.js`, браузер или публичный GitHub‑репозиторий.
3. Храните его как серверную переменную окружения `OPENAI_API_KEY` в Vercel.
4. Рекомендуемый экономичный стартовый вариант — `gpt-5.6-luna`; модель можно изменить переменной `OPENAI_MODEL`.
5. Настройте месячный бюджет и уведомления об использовании в OpenAI Platform.

Локальный шаблон переменных находится в `.env.example`. Реальный `.env.local` исключён из Git.

## 3. Публикация в Vercel

1. Импортируйте GitHub‑репозиторий в [Vercel](https://vercel.com/new).
2. Framework Preset: **Other**; корневая папка — корень репозитория.
3. Добавьте переменные:

```text
OPENAI_API_KEY
OPENAI_MODEL=gpt-5.6-luna
SUPABASE_URL
SUPABASE_ANON_KEY
```

4. Нажмите Deploy.
5. Полученный `https://....vercel.app` добавьте в Supabase Authentication URL Configuration.
6. На телефоне откройте URL и выберите «Добавить на экран Домой».

## Безопасность

- `/api/coach` принимает запросы только от пользователя с действующей Supabase‑сессией;
- OpenAI key остаётся только на сервере;
- RLS разрешает пользователю читать и менять только собственный прогресс;
- сообщения ограничены по длине, а endpoint имеет базовое ограничение частоты;
- медицинские рекомендации ограничены безопасными рамками и не заменяют врача.
